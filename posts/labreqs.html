# Lab Requirements & Setup

Critical: Do NOT Use Cloud/VPS Providers Without Authorization

Using cloud servers (AWS, Azure, DigitalOcean, Linode, Vultr) or VPS providers for security testing is EXTREMELY DANGEROUS without explicit written authorization.

Offensive activities (port scanning, exploit testing, brute-force attempts) and even defensive activities (running IDS, honeypots, analyzing malware samples) can trigger automatic abuse detection systems. This will result in:

  • Immediate account suspension
  • IP blacklisting across multiple providers
  • Legal action from the provider or third parties
  • Loss of all data and services
  • Being permanently banned from the platform

Solution: Build a home lab. Use dedicated hardware you own, on a network you control. This guide assumes you're building a home lab environment.

Home Lab Philosophy

A home lab provides a safe, isolated environment where you can break things, test exploits, simulate attacks, and learn without consequences. You have full control over the network, can snapshot and restore systems instantly, and won't trigger abuse complaints.

Recommended approach: Use an old computer as a dedicated home lab server. This centralizes all your VMs in one place, keeps them isolated from your daily-use machines, and provides a realistic environment for testing network security concepts.

Minimum Hardware Requirements

Option 1: Dedicated Home Lab Server (Recommended)

Best for: Completing all circles, running multiple VMs simultaneously, realistic network testing

Minimum Specifications:

  • CPU: Quad-core (Intel i5/i7 4th gen or newer, AMD Ryzen 3/5/7)
  • RAM: 16GB minimum, 32GB recommended
  • Storage: 500GB SSD (1TB preferred)
  • Network: Gigabit Ethernet
  • Virtualization: CPU with VT-x/AMD-V support (check BIOS)

Why These Specs?

  • RAM: Each VM needs 2-4GB. You'll run 4-6 VMs simultaneously for network testing
  • Storage: Base OS images (20-40GB each) + snapshots (10-20GB each) + log data
  • CPU: Virtualization overhead + running security tools (IDS, SIEM) is CPU-intensive
  • SSD: Dramatically improves VM performance and snapshot operations

Option 2: Your Daily-Use Computer

Best for: Circles 1-3 only, single VM testing, learning basics

Minimum Specifications:

  • CPU: Dual-core with VT-x/AMD-V
  • RAM: 8GB minimum (16GB to run VMs comfortably)
  • Storage: 200GB free space
  • OS: Windows 10/11, macOS, or Linux with virtualization support

Limitations:

  • Cannot run multiple VMs simultaneously for network security testing
  • Performance impact on daily tasks when VMs are running
  • Limited ability to create isolated networks
  • Not suitable for Circles 4+ (Network Security, Blue Team, Red Team)

Virtualization Platform

Choose one based on your host operating system:

VirtualBox (Free, Cross-Platform)

Best for: Beginners, single-user labs, Circles 1-3

Easy to use, good documentation, supports snapshots and virtual networks. Limited performance compared to Type-1 hypervisors.

VMware Workstation Pro / Fusion (Paid, Cross-Platform)

Best for: Advanced users, better performance, Circles 1-4

Superior performance, advanced networking, better snapshot management. Costs $200-300 but offers professional features.

Proxmox VE (Free, Linux-based)

Best for: Dedicated home lab server, all circles, production-like environment

Type-1 hypervisor (bare metal), web-based management, enterprise features. Requires dedicated hardware. Steep learning curve but most realistic.

KVM/QEMU (Free, Linux)

Best for: Linux power users, command-line preference

Native Linux virtualization, excellent performance, full control. Requires Linux host and comfort with command-line management.

    Cost-Effective Hardware Sourcing

    Used Business Workstations: Dell Optiplex, HP EliteDesk, Lenovo ThinkCentre from 2015-2018 (i5/i7, 16-32GB RAM) sell for $100-300 on eBay/Craigslist. Perfect for home labs.

    RAM Upgrades: DDR3/DDR4 is cheap used. Upgrade to 32GB for $50-100.

    Used Servers: Dell PowerEdge R620/R720 with dual CPUs and 64GB+ RAM can be found for $200-500. Loud and power-hungry, but perfect for serious labs.

    Storage & Snapshot Strategy

    Snapshots are critical for security training. You'll break things. A lot. Snapshots let you revert instantly.

    Recommended Snapshot Workflow:

    1. Baseline Snapshot: After initial OS installation and updates. Label: "Fresh Install"
    2. Pre-Configuration Snapshot: Before starting each guide. Label: "Before Part X"
    3. Working Snapshot: After major configuration milestones. Label: "Part X - Working Config"
    4. Broken Snapshot: When something breaks (don't delete!). Label: "Part X - Debugging [issue]"

    Storage Impact: Each snapshot uses 10-30% of VM disk size initially, growing as changes accumulate. Budget 2-3x the base VM size for snapshot storage.

    Network Configuration Tips

    Virtual Network Types:

    • NAT (Network Address Translation): VM gets internet access through host. Use for: VMs that need internet but don't need to talk to each other. Single VM setups.
    • Bridged: VM gets IP on your physical network. Use for: VMs that need to be accessible from other physical devices. Warning: Exposes VM to your LAN - not recommended for attacker VMs.
    • Internal/Host-Only: VMs can talk to each other and host, but not internet. Use for: Isolated test networks, security monitoring networks, attacker VMs. This is your primary network type for Circle 4+.

    Start Small, Scale Up

    Don't let hardware limitations stop you from starting. Complete Circles 1-2 on any computer. Learn the concepts. Build skills. Save up for better hardware as you progress. Many security professionals started with a single VM on an old laptop. The important part is starting and learning consistently.

    "Security is learned through practice, not theory. Break things in your lab so you know how to fix them in production. Every misconfiguration you debug makes you more competent. Every failed command teaches troubleshooting. Build your lab, break it, rebuild it better."
