# Privacy Policy

AnonShell practices what it preaches: absolute privacy by design. We don't collect visitor data because we've engineered our infrastructure to make collection technically impossible.

The Simple Truth: We Have No Logs Because We Can't Create Them

Most privacy policies explain what data they collect and how they "protect" it. This one is different. We don't collect any data, period. Not because we promise to delete it later. Not because we encrypt it well. But because our static site architecture has no server-side processing, no databases, and no logging infrastructure. There's nothing to log because there's no backend to do the logging.

What We Don't Collect (And Can't, Even If We Wanted To)

Zero Data Collection

  • No IP address logging: Static sites on Cloudflare Pages have no server-side logging. Your IP address is never written to any database or log file under our control.
  • No request logging: The pages you visit, the order you visit them, how long you stay - none of this is recorded by us. There's no application server to track requests.
  • No server-side processing: Every page is a static HTML file. No PHP, no Python, no Node.js backend generating personalized content or tracking sessions.
  • No user agent tracking: Your browser type, operating system, and device information are never stored. Static files can't inspect or log request headers.
  • No referrer tracking: We don't log where you came from before visiting AnonShell. There's no backend to receive or store referrer data.
  • No database: AnonShell has zero databases. No SQL, no NoSQL, no key-value stores. Nothing to query, nothing to breach, nothing to subpoena.

Zero Cookies, Zero Tracking Scripts, Zero Surveillance Capitalism

AnonShell serves static HTML, CSS, and JavaScript files. That's it. No tracking pixels. No analytics frameworks. No third-party scripts phoning home to surveillance infrastructure.

What We Use (All Hosted Locally)

  • Tailwind CSS: Frontend framework for styling (local file, no CDN)
  • Highlight.js: Syntax highlighting for code blocks (local file, no CDN)
  • Lucide Icons: Icon library (local file, no CDN)

All JavaScript libraries are served from our own server. No requests leave our infrastructure. No CDN providers see your IP address.

No Cookies = No Consent Banner

You won't see a cookie consent popup on AnonShell because we don't use cookies. Not session cookies. Not authentication cookies. Not tracking cookies. Not "necessary" cookies. No cookies at all.

Your browser visits the site, loads the HTML/CSS/JS, displays content, and that's the end of the relationship. Nothing persists. Nothing tracks. Nothing follows you.

No localStorage, No sessionStorage, No Browser Fingerprinting

AnonShell doesn't use browser storage APIs to track your behavior. We don't store data in localStorage or sessionStorage. We don't implement browser fingerprinting techniques (canvas fingerprinting, WebGL fingerprinting, audio fingerprinting, font enumeration, etc.).

Static Site Architecture: Privacy by Design

AnonShell is deployed as a fully static site on Cloudflare Pages. This architecture choice eliminates entire categories of privacy risks:

Why Static Hosting Guarantees Privacy

  • No application server: Traditional web servers (Apache, Nginx, IIS) log every request. Static hosting serves pre-built HTML files from a CDN with no logging under our control.
  • No server-side code execution: No PHP scripts, no Python applications, no Node.js servers. Every visitor gets identical HTML files. There's no personalization, therefore no tracking.
  • No session management: Sessions require server-side state. Static sites are stateless. We can't track you across pages because there's no backend to maintain session data.
  • No database queries: Database logs are a common surveillance vector. AnonShell has no database, so there's nothing to log, leak, or subpoena.
  • Build-time generation: The site is built once (when we update content), then served as static files. Your visit doesn't trigger any server-side processing.

Even if an attacker compromised our GitHub repository or Cloudflare account, there are no visitor logs to exfiltrate because the architecture cannot create them. You can't steal data that was never collected.

What Happens When You Visit AnonShell

  1. Your browser connects to Cloudflare's CDN over HTTPS (TLS 1.3 encryption).
  2. Cloudflare serves static files (HTML/CSS/JS) from their edge cache. These files are identical for every visitor - no personalization, no server-side generation.
  3. No log entry reaches us. Cloudflare may log requests internally (see section below), but we don't receive, store, or have access to their logs. Our deployment has no logging infrastructure.
  4. Your browser renders the page entirely client-side. The single-page application (SPA) routing happens in JavaScript using the pre-generated navigation.json file.
  5. You navigate the site. Clicking internal links loads content via JavaScript fetch() requests to static HTML files. Everything happens in your browser - no server-side tracking.
  6. You close the tab. Nothing remains anywhere. No session cookies, no localStorage data, no server-side profile, no database entries. It's as if you were never here.

Third-Party Services: None

AnonShell doesn't integrate with any third-party services that could compromise your privacy:

What We Don't Use

  • No Google Analytics: No visitor tracking, no behavior analysis, no advertising profiles.
  • No Facebook Pixel: No social media tracking, no cross-site surveillance.
  • No Cloudflare Analytics: Disabled on our account. We don't receive traffic reports, visitor counts, or request logs from Cloudflare.
  • No third-party CDNs: All JavaScript libraries (Tailwind, Highlight.js, Lucide Icons) are hosted as static files on Cloudflare Pages. No external CDN requests.
  • No comment systems: No Disqus, no Facebook Comments, no third-party embeds.
  • No email newsletter tracking: We don't have a newsletter. No email collection, no open-rate tracking.
  • No payment processors: The site is free. No Stripe, no PayPal, no transaction logging.

Content Delivery: Cloudflare Pages Hosting

AnonShell is hosted on Cloudflare Pages, a static site hosting platform. Here's the complete privacy picture:

What Cloudflare Pages Knows

  • Your IP address: Cloudflare's CDN sees your IP when you request files (unavoidable for any web hosting). However, we don't have access to Cloudflare's logs.
  • Request metadata: Cloudflare logs requests internally (timestamp, user agent, requested path) for DDoS protection and caching. We have disabled Cloudflare Analytics and don't receive any of this data.
  • TLS handshake details: Cloudflare terminates TLS connections at their edge servers to serve cached content efficiently. This is standard for any CDN.

What We Don't Know

  • Who visited the site: We have no IP logs, no visitor database, no analytics dashboard.
  • What pages were viewed: Cloudflare serves files directly from cache. We don't see request logs.
  • When you visited: No timestamps are logged on our end. The static site has no backend to receive timing data.
  • Your location or device: We can't correlate visits, build user profiles, or track behavior patterns because we have zero data.

Why we chose Cloudflare Pages: Static hosting on a CDN eliminates the traditional server (Apache/Nginx) that would log every request. While Cloudflare can see traffic (as any hosting provider must), we have no access to their logs and have disabled all analytics integrations. This is a strict improvement over self-hosted servers where logs exist under our control.

Trust model: You must trust Cloudflare not to abuse their position (same as trusting any ISP, VPN, or hosting provider). However, Cloudflare's privacy policy states they don't sell user data and delete most logs within 24 hours. We've verified our Pages deployment has zero analytics or tracking enabled on our account.

Mitigation: If you don't trust Cloudflare, use Tor Browser to hide your IP and traffic patterns from all network observers. Tor exit nodes can't correlate your visit to your identity, and Cloudflare only sees the exit node's IP.

Government Requests & Law Enforcement

Because AnonShell is a static site with no backend, no database, and no logging infrastructure, we have no visitor data to provide to law enforcement, intelligence agencies, or civil litigants. If served with a subpoena, court order, or National Security Letter demanding:

  • Visitor IP addresses: We don't have them. Cloudflare may have them, but we can't compel Cloudflare to provide data on our behalf (and wouldn't).
  • User accounts or profiles: The site has no user accounts. There's no registration, no login, no authentication system.
  • Viewing history or behavior: Static sites can't track navigation patterns. There's no server-side session to analyze.
  • Private messages or communications: AnonShell has no messaging system, no forms that email us, no contact database.

The honest answer to any data request is: the architecture cannot create that data. It's not that we deleted it or encrypted it beyond recovery - it's that static HTML files served from a CDN have no mechanism to capture visitor information.

Transparency Commitment

If we ever receive a legal request for data, we will publish it in a transparency report (after any gag order expires). However, since we log nothing, there would be nothing to provide even if compelled.

Your Operational Security Is Your Responsibility

While AnonShell doesn't track you, your internet service provider (ISP), VPN provider, browser, and operating system still can. AnonShell's privacy policy only covers what we do (and don't do). To achieve true anonymity:

  • Use Tor Browser to prevent your ISP from seeing what sites you visit.
  • Use a privacy-respecting VPN (like Mullvad) if you prefer VPNs over Tor.
  • Disable JavaScript fingerprinting in your browser settings (uMatrix, NoScript, or Tor Browser's Safest mode).
  • Clear cookies and site data regularly (though AnonShell sets none).
  • Don't log into tracking-heavy services (Google, Facebook) in the same browser session you use to read AnonShell.

Changes to This Privacy Policy

If our privacy practices change (for example, if we add user accounts, comments, or analytics), we will update this page and note the change date at the bottom. The current policy reflects the site's architecture as of December 3, 2025 (migrated to Cloudflare Pages static hosting).

How to Verify Our Claims

  • Inspect the source code: Right-click → View Page Source. Every page is static HTML with no server-side templating.
  • Audit network requests: Open Developer Tools → Network tab. You'll see requests for HTML/CSS/JS files, nothing to analytics domains.
  • Check for cookies: Developer Tools → Application → Cookies. AnonShell sets zero cookies.
  • Examine localStorage: Developer Tools → Application → Local Storage. Should be empty (or only browser defaults).
