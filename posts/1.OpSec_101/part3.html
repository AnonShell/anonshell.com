# 3. OpSec Pre-Flight Checklist

You have read the theory. You understand the threats. Now it is time to act. This checklist breaks security implementation into five progressive levels, from basic hygiene to maximum operational security. Start where you are comfortable, then level up as your threat model evolves.

How to Use This Checklist

  • Start at Level 1 regardless of your technical skill. The fundamentals matter more than advanced techniques.
  • Complete each level fully before moving to the next. Security is cumulative, not modular.
  • Revisit quarterly to verify implementations are still active and up to date.
  • Threat models change. A freelance writer facing data brokers (Level 1-2) has different needs than a journalist in an authoritarian regime (Level 4-5).

Level 1: The Everyday User

Who This Is For: Average internet users who want basic protection against common threats like phishing, credential stuffing, data breaches, and tracking. You use email, social media, online banking, and shopping sites. Your threat model includes opportunistic attackers, data brokers, and mass surveillance.

Time Investment: 2-4 hours initial setup, 30 minutes monthly maintenance.

Level 1 Checklist

Install a password manager (KeePassXC or Bitwarden)
Generate unique passwords (20+ characters) for every account
Enable TOTP-based two-factor authentication on critical accounts (email, banking, social media)
Purchase a hardware security key (YubiKey 5C or SoloKeys) for primary email and financial accounts
Check if your email appeared in breaches at Have I Been Pwned
Enable automatic OS updates (Windows Update, macOS Software Update, APT unattended-upgrades)
Enable automatic browser updates (Chrome, Firefox, Brave)
Install browser privacy extensions: uBlock Origin and Privacy Badger
Enable full-disk encryption (BitLocker on Windows, FileVault on macOS, LUKS on Linux)
Set strong device unlock PINs/passwords (10+ characters, no biometrics for high-value devices)
Enable "Find My Device" and practice triggering remote lock/wipe
Install Cryptomator and create encrypted vaults for sensitive cloud files
Set up encrypted backups with Duplicati or restic
Test backup restoration by recovering a random file
Audit social media privacy settings (set profiles to private, disable location tagging)
Use email aliases (SimpleLogin or AnonAddy) for throwaway signups
Install Signal for encrypted messaging and enable disappearing messages

Level 2: The Power User

Who This Is For: Users managing multiple online identities (personal, work, side projects). You handle sensitive documents, communicate with clients, and want to minimize digital footprint. Your threat model adds targeted phishing, corporate surveillance, and persistent tracking.

Time Investment: 6-10 hours initial setup, 1 hour monthly maintenance.

Level 2 Checklist

Complete all Level 1 items first
Create separate browser profiles for work, personal, and shopping (or use Firefox Multi-Account Containers)
Install VeraCrypt for portable encrypted volumes
Create a VeraCrypt standard volume (not hidden) for sensitive files at rest
Enable encrypted DNS (DNS over HTTPS in browser settings or Quad9 system-wide)
Set up a firewall (Windows Firewall with Advanced Security, macOS Firewall, or GUFW on Linux)
Install OpenSnitch (Linux) or Little Snitch (macOS) for application-level firewall control
Audit installed applications and remove unused software
Disable unnecessary startup programs and background services
Set up GPG keys with GnuPG for email encryption (generate 4096-bit RSA or Ed25519)
Configure email client (Thunderbird) with GPG support
Create email filters to quarantine suspicious attachments
Install metadata removal tools (MAT2 or ExifTool)
Strip metadata from photos before sharing online
Use a VPN for public Wi-Fi (Mullvad or IVPN with audited no-logs policies)
Set up network-wide ad blocking with Pi-hole (optional but recommended)
Rotate online identities annually (new email aliases, update security questions with random answers stored in password manager)
Enable audit logging on sensitive documents (track who accessed what, when)

Level 3: The IT Professional

Who This Is For: IT specialists, system administrators, and technicians handling corporate proprietary data, ERP systems, or unclassified government work. You manage infrastructure, access production systems, and hold privileged credentials. Your threat model includes insider threats, supply chain attacks, and corporate espionage.

Time Investment: 12-20 hours initial setup, 2 hours monthly maintenance.

Level 3 Checklist

Complete all Level 1 and Level 2 items first
Implement LUKS2 full-disk encryption with TPM2 binding (systemd-cryptenroll)
Configure Secure Boot with custom keys (or Heads firmware for advanced users)
Set up VM isolation with QEMU/KVM, VirtualBox, or VMware
Create separate VMs for: work, personal browsing, risky downloads, testing
Enable VM snapshots and practice reverting to clean states
Disable VM shared folders and clipboard (use network file shares with encryption instead)
Install USBGuard to whitelist authorized USB devices
Implement network segmentation (separate VLANs for work, IoT, guests)
Deploy IDS/IPS (Suricata or Snort) on network perimeter
Set up centralized logging with ELK Stack or Graylog
Harden SSH: disable password authentication, use Ed25519 keys, enable fail2ban
Audit system with Lynis and fix critical findings
Enable SELinux (RHEL/Fedora) or AppArmor (Debian/Ubuntu) in enforcing mode
Configure kernel hardening (sysctl settings: disable IPv6 if unused, enable SYN cookies, randomize ports)
Set up WireGuard VPN for remote access to home/work networks
Implement 3-2-1 backup strategy (3 copies, 2 different media types, 1 offsite)
Encrypt all backups with GPG or VeraCrypt before offsite storage
Use hardware tokens for SSH authentication (ssh-keygen -t ed25519-sk with YubiKey)
Implement certificate pinning for critical internal services
Deploy DNSSEC validation with Unbound recursive resolver
Set up tamper-evident seals on physical hardware (glitter nail polish on screws)
Document incident response procedures (what to do if device is compromised)

Level 4: The Cybersecurity Enthusiast

Who This Is For: Self-learners pursuing cybersecurity knowledge, privacy advocates, and individuals wanting military-grade personal security without being persons of interest. You experiment with security tools, run penetration tests on your own systems, and want defense-in-depth. Your threat model includes advanced persistent threats (APT) techniques but not nation-state targeting.

Time Investment: 30+ hours initial setup, 3-4 hours monthly maintenance.

Level 4 Checklist

Complete all Level 1, 2, and 3 items first
Migrate to Qubes OS for Xen-based security through isolation
Configure Qubes disposable VMs for all untrusted browsing
Set up Whonix Gateway + Workstation VMs for Tor-based anonymity
Implement VeraCrypt hidden volumes with plausible deniability
Create decoy data in outer volume, real data in hidden volume
Set up LUKS nuke password to wipe encryption headers under duress
Deploy Coreboot or Libreboot firmware (if hardware supports it)
Disable Intel Management Engine with me_cleaner (or purchase ME-disabled hardware)
Implement air-gapped cold storage for cryptocurrency keys and sensitive GPG keys
Use Tails OS (live amnesic system) for high-risk operations
Configure MAT2 or ExifTool in automated workflows to strip all metadata by default
Set up Tor Browser with custom bridges (obfs4) for censorship circumvention
Implement rotating Tor circuits for different identities
Use Session or Briar for metadata-resistant messaging
Generate offline PGP master key, create subkeys for signing/encryption/authentication
Store master key on air-gapped machine, use subkeys on YubiKey
Implement key rotation schedule (subkeys every 2 years, master key every 5 years)
Set up canary tokens to detect unauthorized access attempts
Deploy honeypot services on network to identify scanning activity
Run regular vulnerability scans with OpenVAS or Nmap NSE scripts
Practice cold boot attack prevention (power off, not sleep; use encrypted swap)
Configure BIOS/UEFI passwords and disable boot from external media
Implement Faraday bag procedures for devices storing sensitive data
Set up kill switch scripts to wipe RAM and dismount encrypted volumes on trigger
Test incident response plan quarterly (simulate compromise, practice recovery)

Level 5: Maximum Operational Security

Who This Is For: Investigative journalists in hostile environments, whistleblowers, activists under state surveillance, and individuals facing nation-state adversaries. You operate under the assumption that all communications are monitored, all devices are potentially compromised, and physical security is paramount. Your threat model includes signals intelligence (SIGINT), human intelligence (HUMINT), and supply chain interdiction.

Time Investment: 50+ hours initial setup, ongoing lifestyle discipline, weekly maintenance.

Critical Warning

Level 5 security requires permanent behavioral changes. You cannot casually browse social media, reuse identities, or trust cloud services. Every action is scrutinized. Mistakes are permanent. If your life or freedom depends on operational security, consider professional security consultation. This level is not theoretical; it is survival protocol.

Level 5 Checklist

Complete all Level 1-4 items first
Operational discipline: assume all communications are monitored, all devices are compromised
Purchase hardware from random retailers with cash, never shipped to your address
Run Qubes OS with Whonix for all internet access (no clearnet browsing from real identity)
Use Tails OS exclusively for high-risk communications (boot from read-only media)
Maintain air-gapped machine (never connected to internet) for key storage and sensitive document creation
Transfer files to/from air-gapped system using one-way data diodes or manual retyping (no USB)
Implement VeraCrypt triple-cascade encryption (AES-Twofish-Serpent) with hidden volumes
Store decoy data in outer volume convincing enough to satisfy coercion
Use diceware passphrases (10+ words, 128+ bits entropy) for critical encryption keys
Memorize all critical passwords; never write them down digitally or physically
Implement dead man's switch for automatic data release if you are detained/disappeared
Disable all biometrics (fingerprint, face unlock); use strong PIN/password only
Apply glitter nail polish to all hardware screws for tamper detection
Photograph hardware serial numbers; verify on each boot
Store devices in locked containers when not in use; never leave unattended
Use Faraday bags for phones when traveling; remove batteries if possible
Purchase burner phones with cash; register with fake identities; rotate every 30 days
Never connect burner phone to home Wi-Fi or use near regular devices
Communicate exclusively through Signal with disappearing messages (7 days max)
Verify safety numbers in person for all Signal contacts before sensitive discussions
Use Session (onion-routed messaging) or Briar (Tor + Bluetooth mesh) for highest anonymity
PGP-encrypt all emails; use ProtonMail or Tutanota with Tor access
Implement key ceremony procedures (witnesses, multiple signing parties) for critical key generation
Split GPG master key using Shamir's Secret Sharing (require 3 of 5 shares to reconstruct)
Store key shares in geographically distributed safe deposit boxes (different banks, different cities)
Run all internet traffic through Tor (Whonix Gateway or transparent Tor proxy)
Use Tor bridges (obfs4, meek) to hide Tor usage from network observers
Chain VPN → Tor → VPN for multi-hop anonymity (different jurisdictions, cash-paid VPNs)
Rotate Tor identities every 10 minutes; never reuse same identity across sessions
Use separate hardware for each identity (work, personal, pseudonymous)
Never mix identities on same device or same network
Rewrite all writing with LLM tools to remove stylometric fingerprints
Translate text through 3 languages before publishing (original → language A → language B → back to original)
Randomize sentence length, vocabulary complexity, punctuation patterns
Strip all metadata from documents (ExifTool, MAT2) before upload
Print documents, photocopy, scan back in to remove digital forensic artifacts
Use LibreOffice with randomized fonts/spacing instead of Microsoft Office (unique document fingerprinting)
Implement secure file deletion (shred -vfz -n 7 or DBAN for full disk)
Encrypt swap partition with random key on each boot
Disable core dumps, crash reports, and hibernation (prevent RAM leakage)
Set up network kill switch to block all traffic if VPN/Tor disconnects
Deploy MAC address randomization on every network connection
Use public Wi-Fi from coffee shops, libraries (never home network for sensitive ops)
Sit with back to wall, screen not visible to cameras or people
Use privacy screen protectors to prevent shoulder surfing
Conduct sensitive communications in noise-generating environments (busy cafes, public transit)
Practice counter-surveillance: detect if you are being followed or monitored
Maintain cover story for all activities; never discuss real operations
Establish duress code words with trusted contacts (signal compromise without alerting adversary)
Document emergency procedures: who to contact, how to destroy evidence, where to flee
Keep go-bag ready (cash, burner phone, encrypted USB, emergency contacts)
Test full operational security quarterly: assume compromise, execute destruction plan, verify nothing recoverable

Final Thoughts

Security is not a destination. It is a continuous process of assessment, implementation, and refinement. Most people will never need Level 5 security. Most will be well-served by Level 1-2. The key is matching your defenses to your actual threats, not your imagined ones.

Remember

  • Perfect security does not exist. You are mitigating risk, not eliminating it.
  • The weakest link is usually human behavior, not technical controls.
  • Complexity is the enemy of security. Do not implement what you cannot maintain.
  • Test your defenses regularly. Untested security is wishful thinking.
  • Share knowledge freely. Collective security benefits everyone.
"Security is not a product, but a process. It is not something you buy, but something you do."
Bruce Schneier

Start small. Build habits. Level up when ready. The adversaries are patient. So must we be.
